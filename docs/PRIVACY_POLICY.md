# Политика конфиденциальности

**Telegram-бот «Помощник ВкусВилл»**
Дата вступления в силу: 14 февраля 2026 г.
Последнее обновление: 14 февраля 2026 г.

---

## 1. Общие положения

Настоящая Политика конфиденциальности описывает, какие данные
обрабатывает Telegram-бот «Помощник ВкусВилл» (далее — «Бот»),
как они используются, кому передаются и как вы можете управлять
своими данными.

Бот является **независимым открытым проектом** (open source) и
**не является** официальным продуктом компании ВкусВилл. Бот
использует открытый API ВкусВилл для поиска товаров и формирования
корзины.

Продолжая использование Бота, вы соглашаетесь с условиями настоящей
Политики.

---

## 2. Какие данные мы обрабатываем

### 2.1. Данные, получаемые автоматически

| Данные | Цель обработки | Срок хранения |
|--------|---------------|---------------|
| Telegram ID (числовой) | Идентификация пользователя в Боте | Бессрочно (до удаления по запросу) |
| Языковой код Telegram | Определение языка интерфейса | Бессрочно |

**Мы НЕ сохраняем:** имя, фамилию, username, фото профиля и другие
персональные данные из Telegram-аккаунта.

### 2.2. Данные, предоставляемые пользователем

| Данные | Цель обработки | Срок хранения |
|--------|---------------|---------------|
| Текст сообщений | Генерация ответов с помощью ИИ-модели | Временно (сессия диалога, до /reset или автоочистка) |
| Продуктовые предпочтения | Персонализация подбора товаров (по команде «запомни») | До удаления пользователем |

### 2.3. Данные, генерируемые в процессе использования

| Данные | Цель обработки | Срок хранения |
|--------|---------------|---------------|
| История диалога | Контекст текущей беседы | Временно (Redis/память, автоочистка по LRU) |
| События использования | Аналитика качества сервиса (анонимизированная) | 90 дней |
| Ссылки на корзину | Формирование заказа на сайте ВкусВилл | Временно (24 часа) |

## 3. Как обрабатываются данные

### 3.1. ИИ-модель (GigaChat)

Текст ваших сообщений передаётся в **GigaChat** — ИИ-модель компании
**ПАО Сбербанк** (ИНН 7707083893) — для генерации ответов. GigaChat
обрабатывает текст в реальном времени для:

- Понимания запроса (какие продукты нужны)
- Подбора товаров из ассортимента ВкусВилл
- Расчёта рецептов и ингредиентов
- Формирования текстового ответа

Данные передаются через защищённый канал (HTTPS/TLS). Обработка
регулируется [Условиями использования GigaChat](https://developers.sber.ru/docs/ru/policies/gigachat-terms).

### 3.2. Поиск товаров (MCP API ВкусВилл)

Поисковые запросы к товарам передаются на сервер ВкусВилл через
протокол MCP (Model Context Protocol). Передаётся **только поисковый
запрос** (название продукта), без идентификатора пользователя.

### 3.3. КБЖУ (Open Food Facts)

При запросе информации о калорийности используется открытая база
**Open Food Facts** (openfoodfacts.org). Передаётся **только название
продукта**, без идентификатора пользователя.

### 3.4. Логирование

Логи работы Бота сохраняются для мониторинга и отладки:

- **Персональные данные автоматически маскируются:** телефоны, email,
  номера карт, ИНН, СНИЛС заменяются на `[PHONE]`, `[EMAIL]`,
  `[CARD]` и т.д.
- **Telegram ID хешируется** (SHA-256) в системах аналитики —
  идентифицировать пользователя по хешу невозможно.
- Логи хранятся **не более 90 дней** и автоматически удаляются.

### 3.5. Аналитика (LLM-observability)

Для улучшения качества ответов используется система аналитики Langfuse:

- Telegram ID хешируется (SHA-256)
- Содержимое сообщений может быть полностью скрыто (настройка оператора)
- Данные используются **исключительно** для анализа качества ИИ-ответов

## 4. Кому передаются данные

| Получатель | Какие данные | Цель |
|-----------|-------------|------|
| **ПАО Сбербанк** (GigaChat API) | Текст сообщений | Генерация ответов ИИ |
| **ВкусВилл** (MCP API) | Поисковые запросы (названия продуктов) | Поиск товаров, формирование корзины |
| **Open Food Facts** | Названия продуктов | Информация о КБЖУ |
| **Yandex Cloud** | Логи (с маскировкой PII) | Хранение логов |
| **Langfuse** | Анонимизированные трейсы | Аналитика качества |

Данные **не продаются** третьим лицам и **не используются** для рекламы.

## 5. Защита данных

Мы применяем следующие меры защиты:

- **Минимизация данных:** собираем только Telegram ID и языковой код;
  имя, фамилия и username не сохраняются.
- **Маскировка PII:** персональные данные в текстах (телефоны, email,
  карты) автоматически маскируются перед логированием.
- **Хеширование:** Telegram ID хешируется (SHA-256) в аналитике —
  обратное восстановление невозможно.
- **Шифрование каналов:** все внешние API используют HTTPS/TLS.
- **Автоудаление:** логи хранятся не более 90 дней; история диалога
  очищается автоматически.
- **Rate limiting:** защита от злоупотреблений (5 сообщений / 60 секунд).
- **Исходный код открыт:** вы можете проверить, как обрабатываются
  данные, в публичном репозитории проекта.

## 6. Ваши права

### 6.1. Удаление истории диалога

Отправьте команду `/reset` — история текущего диалога будет
немедленно удалена.

### 6.2. Удаление предпочтений

Вы можете удалить сохранённые предпочтения, написав Боту
«удали предпочтение [категория]» (например, «удали предпочтение молоко»).

### 6.3. Полное удаление данных

Для полного удаления всех данных (Telegram ID, события, предпочтения)
свяжитесь с оператором Бота. После получения запроса данные будут
удалены в течение 30 дней.

### 6.4. Отказ от использования

Вы можете прекратить использование Бота в любой момент, заблокировав
его в Telegram. После блокировки новые данные не будут собираться.
Для удаления ранее собранных данных используйте пункт 6.3.
