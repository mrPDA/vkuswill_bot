# Правила для агентов

Правила в формате `.mdc` автоматически загружаются Cursor AI.

## Режимы загрузки

- **`alwaysApply: true`** — загружается в контекст КАЖДОГО запроса (дорого по токенам, использовать только для критичного)
- **`alwaysApply: false`** — загружается автоматически, когда `description` совпадает с контекстом запроса

Все правила этого проекта используют `alwaysApply: false` — Cursor подключает их по ситуации.

## Правила

| Файл | Когда подключается | Строк |
|------|-------------------|-------|
| `devops-agent.mdc` | git-операции: коммит, push, PR, релиз, ветки | 301 |
| `git-commits.mdc` | создание коммитов | 65 |
| `git-branches.mdc` | создание и именование веток | 72 |
| `git-pull-requests.mdc` | создание Pull Request | 117 |
| `git-code-review.mdc` | проведение code review | 161 |
| `git-versioning.mdc` | релизы и версионирование | 198 |
| `testing-agent.mdc` | тестирование, безопасность, SAST | 178 |

## Иерархия

```
devops-agent.mdc          ← Операционный воркфлоу (КАК и КОГДА)
├── git-commits.mdc       ← Справочник: формат коммитов
├── git-branches.mdc      ← Справочник: именование веток
├── git-pull-requests.mdc ← Справочник: структура PR
├── git-versioning.mdc    ← Справочник: SemVer и релизы
├── git-code-review.mdc   ← Справочник: процесс ревью
└── testing-agent.mdc     ← Справочник: тестирование
```

`devops-agent.mdc` — главное правило, содержит полный воркфлоу. Остальные — детальные справочники по отдельным аспектам.
